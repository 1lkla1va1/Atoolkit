## 逐条并排对照

### ① P1 · 垃圾洞清单去逐条软例外
| 现行 v1 | v2 |
|---|---|
| 下列发现**绝对不报**，除非你能给出…完整 PoC： | 下列发现**绝对不报**。唯一例外：…完整窃取/利用 PoC。（例外只有这一句，逐条不再写"除非"，命中即丢）⚙ |
| - CORS —— **除非证明窃取了具体数据** | - CORS —— 跨域本身不是漏洞 |
| - SSL/TLS —— **除非是严重降级攻击** | - SSL/TLS —— 配置项，不是结果 |
| - 单独的开放重定向 —— **无法链式利用时几乎无害** | - 单独的开放重定向 —— 单独存在无害（可链式利用属"结果"→去决策树） |
| - 目录列举/默认页/报错堆栈 —— 现象，**除非据此拿到结果** | - 目录列举/默认页/报错堆栈 —— 现象，不是结果 |

> 取舍说明：开放重定向不是简单删——"单独存在无害、可链式利用属于结果"，所以把链式利用从"垃圾洞例外"挪进**决策树**当攻击面，而非留在清单里当"除非"。这正是"现象 vs 结果"的切法。
