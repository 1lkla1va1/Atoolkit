# Atoolkit v9.0.1 迭代方案：实跑入口与诊断闭环修复

日期：2026-07-20

## 1. 背景与裁定

v9.0.0 的确定性合同通过了仓库回归，但 2026-07-19 两次 shop 黑盒实跑没有进入该合同：实际 Codex workspace 使用了旧版外层 `AGENTS.md`，全局 `/src` 仍指向其他项目；两个 Run 均缺 Manifest/Plan、Inventory、Threat、Execution、Attribution、Agenda、Receipt，21 个 Finding 的 proof accepted 数为 0，Agent 仍直接创建了 `final_report.md`。

本补丁的目标不是放宽 v9.0 authority，也不是自动把旧 Finding 修成 accepted，而是保证：

1. 实际运行入口不是当前版本时，在首个目标网络动作前失败；
2. 空 Inventory/空 Coverage/内存中的真空归因不能通过实跑审计；
3. 手工终态、宽权限凭据和保留报告文件能被确定性发现；
4. 指定外部 validator 输出时默认不污染被审计 Run；
5. Direct checkpoint 把 proof rejection 的具体原因和保留产物违规反馈给 Agent；
6. 真实 Recon 中已经观察到的端点必须进入 Inventory/Cell 回归；
7. 两次历史实跑继续保持 legacy/unverified，不被原地升级。

## 2. 非目标

- 不把 Direct/Qoder 或当前 POSIX process-group wrapper 标为 `authority_trusted=true`；
- 不实现或伪造 cgroup/container/job supervisor attestation；
- 不自动接受、迁移或重写历史 Finding；
- 不因 oracle 已知而加入靶场端点或漏洞特判；
- 不允许 Audit/Attribution 自身关闭 Coverage；
- 不让外部 `/src` 配置成为显式 Python/Engine 入口的隐式信任来源。

## 3. 修复设计

### 3.1 活跃 Workspace 指令绑定

新增统一的 workspace instruction inspection：

- 对显式 workspace 的 `AGENTS.md` 做 no-follow 常规文件检查；
- 与当前仓库生成后的 `AGENTS.md` 做 SHA-256 精确比较；
- 输出 `ok/missing/invalid/drift`，绝不自动覆盖；
- `run.py --doctor` 默认检查当前工作目录；可用 `--workspace-root` 显式指定；
- Engine live 启动使用同一 strict gate；`--self-check`/`--dry-run` 不产生目标网络动作，因此保持离线测试兼容；
- strict doctor 遇 workspace drift 或 foreign `/src` 返回非零；只有显式 `--allow-foreign-src` 可承认不使用该别名；
- `engine.skill_runtime preflight/init` CLI 必须绑定 workspace 指令；无法推断 workspace 或 hash 不一致时在创建网络前状态前失败；
- `runtime-preflight.json` schema 升级并记录 repo/workspace instruction path 与 digest，但继续固定 `authority_trusted=false`。

Direct library API 保留显式的非严格参数，供离线单测和上层可信 Host 组合；面向用户的 CLI 默认严格。

### 3.2 Audit 真值修复

八项标准不再使用“文件名存在”或“零对象归因 complete”作为充分条件：

- `no_silent_omission` 要求 Manifest、Run Plan、非空 Inventory、非空 Coverage、落盘且一致的 Attribution；
- `exact_miss_attribution` 要求非空 Inventory/Coverage、物理 `miss-attribution.json` 存在、`complete=true`，并与只读 reducer 投影一致；
- `automatic_next_run` 要求物理 Agenda 与 Attribution digest/count 一致；
- 手工报告中出现 `VULN_FOUND`、`LOW_ROI`、显式 complete/全域覆盖声明且 submission 不合格时，标记 manual terminal claim；
- 凭据扫描扩展到整个 Run。含敏感内容或 credential-like 文件名的文件若 group/other 可读写执行，则标记不安全；不输出秘密值；
- `run_plan.json` 纳入 required artifact；外部 authority plan 只能通过有效 Manifest/Receipt 证明，不靠散文豁免。

### 3.3 Validator 写入边界

`validate_run_artifacts()` 增加独立 `write_sidecars` 控制：

- 默认输出在 Run 内时，继续写 `finding_validation.json`、Attribution、Agenda；
- 显式 `--output` 指向 Run 外时，默认只写主输出，不修改 Run；
- 只有显式 `--write-sidecars` 才允许同时写 Run 内 sidecar；
- `audit` 继续使用 `write_output=false`，保证目录前后完全一致。

### 3.4 Direct Proof 反馈与保留产物门

Checkpoint 增加：

- bounded rejected finding 明细：ID、相对路径、规范化 method/endpoint/param/role/class、reasons；
- `proof_repair_required` 数量；
- `reserved_artifact_violations`：Agent 手工创建 `final_report.md`、`summary.json`、`delivery_status.json`、`submission_status.json`；
- 任一保留产物或 rejected proof 存在时 `report_ready=false`；
- runtime queue 明确优先提示修复 canonical proof，不把 Markdown Finding 当结果。

不自动改写或删除 Agent 产物，避免诊断器成为新的隐式真值写入者。

### 3.5 Recon → Inventory 回归

不增加靶场特判。新增通用 HTML/JS fixture，验证：

- `fetch(...?user_hash=...)` 生成 GET/query param；
- `fetch(..., {method: 'POST', body: JSON.stringify({order_time, use_points})})` 生成 POST/body params；
- Direct init 生成非空 Inventory、Coverage 与 Execution Queue；
- 空 Recon 必须失败，不能用空 ledger 进入攻击阶段。

### 3.6 本地安装修复

- 同步外层 workspace `AGENTS.md`，保留独占备份；
- 将当前用户 `/src` 别名切换到本仓库 `codex/prompts/src.md`；
- 安装动作必须显式执行、可回滚，不由 doctor 自动修改；
- 更新 README 启动顺序：doctor(strict) → preflight → recon → init → observe/checkpoint → finalizer/submission。

## 4. 兼容与迁移

- Manifest/ProjectState/Finding schema 不升级；
- runtime preflight schema 升级仅影响新 Direct session，已有 session 保持只读审计；
- `doctor()` library 默认保持非 strict，CLI 使用 strict，避免破坏旧嵌入调用；
- validator 仅改变“外部 output”的副作用默认值；Run 内默认路径兼容原 finalizer；
- 历史两次 shop Run 不生成 Manifest、不写 Attribution、不升级 Finding。

## 5. 发布门

1. 新增失败先行测试全部通过；
2. 全量 pytest 通过；
3. self-check、doctor、regen、compileall、diff-check 通过；
4. strict doctor 对旧 workspace/foreign `/src` 失败，对修复后环境通过；
5. 两次历史 Run audit 均为 issues_found，且目录 hash/文件清单前后不变；
6. 第一轮必须显示空分母失败、manual terminal claim、unsafe credential；第二轮必须显示缺 ledger/attribution/receipt；
7. 外部 validator output 不在历史 Run 内产生任何文件；
8. 通用 Recon fixture 能把已观察端点转为非空 cell；
9. 版本升级为 9.0.1，并同步 engine/SKILL/Changelog/生成指令。
