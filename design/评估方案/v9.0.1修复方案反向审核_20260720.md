# Atoolkit v9.0.1 修复方案反向审核（2026-07-20）

审核对象：`design/迭代方案/迭代方案v9.0.1_实跑入口与诊断闭环修复.md`

## 1. 第一轮威胁审查

若直接按“让实跑通过”修改，最容易引入以下反向风险：

1. 为了让 Direct 报告可提交而放宽 authority；
2. doctor 自动覆盖用户 workspace 指令或全局 `/src`；
3. validator 为了生成 Attribution 再次污染 legacy Run；
4. 把空 ledger 的零对象归因当成完整；
5. audit 扫描凭据时把秘密值打印到 JSON；
6. 自动修补历史 Finding，形成无人工复验的 accepted truth；
7. 为 shop 靶场硬编码 endpoint/payload，制造虚假能力提升；
8. strict workspace 检查破坏所有 library fixture 和外部可信 Host 组合；
9. 只增加 Prompt，不增加机器门，下一次仍可重复生成手写 final；
10. 只看 Finding 数，忽略 Recon 已发现但未进入 Inventory 的 seam。

原方案只有在逐项封闭这些风险后才可施工。

## 2. 复核结论

### 2.1 Authority 不放宽

方案明确保留 `authority_trusted=false`，不声称已实现容器/网络 pre-exec enforcement。通过。

### 2.2 安装与诊断职责分离

doctor/preflight 只读检查并 fail closed；同步 workspace 和 `/src` 由显式安装步骤执行，保留备份。通过。

### 2.3 Legacy Run 不变性

Audit 使用 `write_output=false`；外部 validator output 默认不写 sidecar；历史 Finding 不自动修复。通过。

### 2.4 空分母不再真空通过

标准要求非空 Inventory、非空 Coverage 和物理 Attribution 三者同时成立，避免 `all([])` 语义。通过。

### 2.5 凭据扫描不泄密

只输出 path、kind、mode，不输出匹配文本；仅以权限和路径策略定性。通过。

### 2.6 无靶场特判

Recon 回归使用通用 fetch/jsonPost fixture，历史 Run 只作为只读验收输入。通过。

### 2.7 Strict CLI 与 Library 兼容

面向用户的 CLI 默认 strict；library API 由调用者显式请求 strict，使 dry fixture/可信 Host 不被隐式破坏。通过。

### 2.8 Proof 反馈不升级真值

Checkpoint 只返回 rejection/repair 信息，不改 Finding、不写 final report、不把 rejection 变 accepted。通过。

## 3. 失败先行测试清单

施工前必须新增或更新以下测试：

1. strict doctor：workspace AGENTS drift → `ok=false`；
2. strict doctor：foreign `/src` → `ok=false`，显式 acknowledge 后可通过；
3. Direct CLI preflight：无 workspace AGENTS 或 digest drift → exit 2，且不创建 preflight；
4. Direct preflight：匹配指令 → schema 2，记录 digest，保持 untrusted；
5. audit：空 coverage、无 inventory 的 legacy Run 不得通过 `no_silent_omission/exact_miss_attribution`；
6. audit：手写 final 中含 `VULN_FOUND` 必须标 manual claim；
7. audit：run 根/evidence 下 0644 cookie 被发现，0600 identity store 不误报；
8. validator：外部 `--output` 前后 Run 文件清单一致；
9. validator：默认 Run 内输出仍写 validation/attribution/agenda；
10. checkpoint：rejected Finding 返回原因，手工 `final_report.md` 阻断 report_ready；
11. Recon：通用 fetch/jsonPost 转换为 method/param 完整的 inventory/cell；
12. 历史 shop Run audit 只读且保持 unverified；
13. v9.0 全部回归继续通过。

## 4. 最终审核裁定

**APPROVE FOR IMPLEMENTATION**，批准范围仅为：

- workspace/runtime instruction binding；
- strict doctor 与显式 alias acknowledgement；
- audit 真值与凭据权限修复；
- validator sidecar 写入边界；
- Direct checkpoint proof/reserved-artifact feedback；
- 通用 Recon 闭环回归、文档、版本和本地显式安装同步。

不批准：

- 放宽 authority/delivery/submission；
- 自动修历史 Finding；
- 靶场特判；
- 自动覆盖用户配置；
- 用新增 Attribution 让 legacy Run 变 complete；
- 未经 receipt 的 final report。
