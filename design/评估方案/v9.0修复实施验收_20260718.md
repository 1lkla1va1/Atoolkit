## 3. 关键负向验收

1. 手写或篡改 `final_report.md`：submission non-zero；
2. 报告含 Cookie/Token/PII：secret scan 拒绝且不输出原值；
3. 只有 HTTP 500/type confusion：Finding 不具备提交资格；
4. 当前用户凭据回显但未证明跨边界：拒绝；
5. proof-invalid Finding：不进入 registry，只生成 Host proof-repair continuation；
   同批其他单条合格但被原子门阻断的 Finding 也各自生成带目标端点的 repair 归因；
6. incomplete Run：不提交 negative/dead-end/coverage closure，只提交 Host continuation 和独立通过 proof 的 root；
7. 普通模型 Intent 指向 inventory 外路径：scheduler 拒绝；Host continuation 可在下一 Run 恢复；
8. 后续阴性与 confirmed cell 冲突：confirmed 不被覆盖，Fact/Finding 转强制复验并生成 critical continuation；
9. Manifest/authority 损坏：连 continuation 也不得提交；
10. Direct/legacy 无 Manifest：最多生成 `legacy_draft_report.md`，不能占用 Canonical 文件名。
