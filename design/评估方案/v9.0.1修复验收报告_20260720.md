# Atoolkit v9.0.1 修复验收报告

日期：2026-07-20

## 1. 验收结论

v9.0.1“实跑入口与诊断闭环”修复通过验收，可以阻断本次 v9.0 实跑暴露出的主要工程失效路径：旧工作区指令/外来 `/src`、空覆盖真空通过、内存归因冒充落盘真值、外部验证输出污染历史 Run、Direct proof rejection 不反馈、Agent 手写 finalizer 保留产物以及凭据文件权限风险漏检。

该结论仅代表本补丁范围完成，不代表 Atoolkit 已成为 authority 完全体。当前内置 Codex backend、POSIX wrapper 和 Direct Skill 仍不能证明 cgroup/container/job 级的后代进程收口与 pre-exec 出站约束，因此继续固定 `authority_trusted=false`，不能自动提交 ProjectState 或声称 verified delivery。

## 2. 方案审核结果

反向审核批准 v9.0.1 patch scope，并明确拒绝以下捷径：放宽 Finding proof schema、把人工 Markdown 转成 accepted Finding、用空 ledger 计算 100% 覆盖、Audit 原地补写 Attribution、添加 shop/oracle 特判、将 Direct/wrapper 标为可信。实施结果遵守这些限制。

## 3. 修复项验收

1. **活跃指令绑定**：严格 doctor 对当前 workspace `AGENTS.md` 做 no-follow 与项目 SHA-256 精确校验；Engine live 和 Direct CLI 在首个目标网络动作前失败关闭。修复后本机 workspace 状态为 `ok`，项目/compatibility/workspace digest 均为 `19e15aa46de7d7cac3b8a0c0154dd927810c2c00d634935e9a5197aa17158c16`。
2. **`/src` 归属**：`~/.codex/prompts/src.md` 已从其他项目切换到本项目 `codex/prompts/src.md`；strict doctor 返回 `src_alias.status=project`。
3. **Audit 真值**：`run_plan.json` 纳入必需产物；Inventory/Coverage 必须非空；Attribution/Agenda 必须物理存在并与只读 reducer 投影一致；空分母不得通过 `no_silent_omission`、`no_false_coverage` 或 `exact_miss_attribution`。
4. **人工终态识别**：不合格报告中的 `VULN_FOUND`、`LOW_ROI`、complete/全域覆盖声明会触发 `manual_complete_claim_without_verified_delivery`。
5. **敏感文件权限**：扫描扩展到整个 Run，只输出路径、类型和权限，不输出秘密值；group/other 有权限的 Cookie、认证头、秘密/PII 证据会触发问题。
6. **Validator 写边界**：外部 `--output` 默认不再生成 Run 内 `miss-attribution.json`/`next-run-agenda.json`；显式 `--write-sidecars` 才允许该行为。
7. **Direct 反馈**：checkpoint 输出 bounded rejected proof 明细、`proof_repair_required` 和 finalizer 保留产物违规；任一存在时 `report_ready=false`。
8. **通用 Recon**：回归 fixture 能从普通 HTML/JS 中恢复 `GET /api/user/balance-records.php?user_hash=...` 以及 `POST /api/user/create-order.php` 的 `product_no/quantity/order_time/use_points` body 参数；未添加靶场特判。
