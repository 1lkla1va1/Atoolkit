## 第一轮：REJECT

两路审核一致认为初稿方向正确，但存在会导致“测试绿、真实 Run 仍自证或失控”的阻断项：

1. 当前 CodexAdapter 无法在嵌套 `codex exec` 的内部 shell 发包前拦截，不能把一个 URL policy 函数描述成 hard egress。
2. 没有冻结本轮 scope，closure 可能要求全项目闭合或允许任意漏项。
3. CellIdentity 未完整表达 origin/base path、param location 和 IDOR actor/owner/object boundary。
4. proof-pass/closure-fail 的逐类提交白名单不明确。
5. finalizer 只是多文件顺序写，不具备 exactly-once、WAL、CAS、并发和崩溃恢复语义。
6. receipt 只有可重算 digest，缺 immutable snapshot、commit chain 和模型不可写 authority anchor。
7. Skill Mode authority 仍可能在模型可写域，共享 CLI 不等于可信收尾。
8. closure 只看最终 session JSON，模型删 inventory/candidate 行可缩小分母。
9. safe I/O 未覆盖 parent symlink、TOCTOU 和全量权威写入点。
10. unresolved method 没有可执行队列；旧 shop Run 也没有 legacy migration 闭环。
11. 成功标准错误依赖“多 Run 必须新增漏洞”，报告同源投影与冲突验收不足。

因此初稿被正式驳回，没有提前进入生产代码修改。

## 第二轮：一方 APPROVE，一方定点 REJECT

修订版补齐：

- `deny-live` 默认与 unrestricted 明示降级，不再虚构 pre-exec enforcement；
- authority-owned frozen `run_plan`、monotonic discovery/candidate events、`run_complete/project_complete`；
- versioned CellIdentity v2 与 golden vectors；
- proof/closure outcome 提交白名单；
- WAL/lock/CAS exactly-once finalizer；
- immutable before/after snapshots、commit/receipt anchor hash chain；
- Engine/Wrapped Skill/Direct Skill 三种信任等级；
- dirfd/nofollow safe I/O；
- schema v2 与 legacy shop transaction migration；
- 允许零新增漏洞的可重算评估指标。

真实靶场审计方给出 APPROVE。安全审计方仍指出两个精确缺口：

1. 缺 `proof=pass / closure=fail / findings=0` 终态与 revision 语义；
2. 仍可能根据 target 尾部 `/` 推断 base path，`/login/` 会破坏跨 Run identity 稳定性。

## 定点修订与终审

最终方案增加：

- 零 finding + closure fail 固定为 `incomplete / exit 2`，不提交任何 project truth、不增加 revision，只写 authority diagnostic event；
- base path 只接受显式 `--base-path`，并由 authority manifest 记录 `base_path_explicit=true`；target path/trailing slash 永不作为 namespace 推断信号；
- 对应 revision 与跨入口稳定身份测试。

安全审计方定点终审 APPROVE，冻结 hash 如上。两路审核均确认无剩余方案级阻断项。

## 实施约束

APPROVE 只代表方案可施工，不代表代码自动通过。实施必须逐项落实方案中的机器验收，尤其是：

- authority snapshot 读取也必须逐级 nofollow 且只接受 regular file；
- unrestricted network assurance 与 delivery/exit 映射固定且不可误称 preexec；
- crash point 与并发 finalizer 的 exactly-once 故障注入；
- session inventory/candidate 删行不能缩小 closure；
- summary、final report、benchmark 必须是同一 validation+commit 的纯投影。
