## 3. 关键实现与迁移行为

1. ProjectState schema 2 → 3 时创建 `project_state.pre-v91.schema2.json` 私有备份；旧聚合 cell 保留原 status/evidence，但标记 `stale_requires_retest`，不得猜分并闭合新 exact sibling。
2. continuation 的三个 prior sidecar 均采用 no-follow、bounded read；重算 validation digest 与确定性 agenda，篡改、跨 origin、method 冲突、非法路径全部拒绝。
3. 直觉探索每个方向必须有稳定 ID、理由以及真实 HTTP request/response 证据；证据 hash 进入 validation artifact。
4. 跨阶段阴性只接受由 evidence packet 派生的 encoding/strategy family；声明值与派生值不一致会被拒绝。
5. 浏览器真实 fetch/form 的 GET 语义保留；禁止的是覆盖身份层面对未知 method 的猜测，不影响实际 HTTP request builder。
