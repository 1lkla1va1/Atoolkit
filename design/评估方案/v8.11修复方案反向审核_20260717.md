# Atoolkit v8.11 修复方案反向审核（2026-07-17）

审核对象：`design/迭代方案/迭代方案v8.11_威胁建模真值与Canonical交付.md`

说明：这是实施前的反向设计审核，目标是优先寻找会破坏 authority、制造虚假覆盖、
泄露凭据或让垃圾报告继续绕过 validator 的问题。审核不把同一会话冒充独立第三方。

## 第一轮结论：REJECT

初稿存在四个阻断问题：

1. threat cell 有 `threat_id`，但 Finding 合同没有要求绑定 threat，仍可能用同一路径上
   另一个漏洞的证据串格；
2. 可信 wrapper 运行中动态发现新 endpoint 后如何处理未定义，若直接追加 cell 会破坏
   host-owned frozen run plan；
3. finalizer 当前 snapshot 会收集 run 内普通文件，初稿没有明确排除模型预写的
   `final_report.md`，手写报告可能进入 authority 输入树；
4. manifest 的 execution settings 没有限定字段，直接记录 adapter/命令参数可能泄露
   Cookie、Token、代理凭据或环境变量。

因此初稿不准施工。

## 修订复核

### 1. Threat 与 Finding 证据绑定

修订后 threat-mode Finding 必须提供 `feature_point.feature_id` 与 `claim.threat_id`，
validator 只允许它关闭同一编译 threat 的 exact cell。

结论：通过。

### 2. Frozen plan 与动态发现

修订后 trusted run 的新发现只进入 `next_run_discoveries`/pending Intent，不改变本轮
authority plan。Direct diagnostic 可以显示候选，但不能关闭未冻结 cell。

结论：通过。

### 3. Canonical report 输入隔离

修订后 final/draft report 明确属于 derived projection，必须排除出 snapshot 输入；
finalizer 只从 authority snapshot 的 proof-confirmed Finding 重新渲染。

结论：通过。

### 4. Provenance 秘密边界

修订后 provenance 采用固定 allowlist，只记录 provider/model/adapter 和少量非秘密
推理设置；禁止环境变量、凭据和任意 settings，并纳入 manifest replay identity。

结论：通过。

### 5. 兼容性与“完全体”措辞

风险：Engine v8.11 仍使用 `legacy_risk` 单阶段计划，不能被称为完整 threat-model
执行路径。

方案已明确：legacy Engine 记录 `planning_degraded=true`，v8.11 只交付 threat contract、
Direct/wrapper 编译路径和 canonical delivery；Engine 两阶段 host freeze 属于 v8.12。
文档和最终验收不得把 v8.11 宣称为终局 v9。

结论：范围内通过。

### 6. Discovery gate 真实性

风险：六个 `covered=true` 仍可能只是模型自报。

修订后 covered channel 必须引用 run 内真实证据，validator 使用 no-follow 路径读取并
校验存在性；blocked/N/A 必须给 reason。

结论：通过。

### 7. Threat model 自身可能漏威胁

这是黑盒威胁建模的剩余风险，无法由 schema 完全消除。当前控制包括：全 inventory
feature 归属、六渠道证据、coverage note、无 threat 理由、unruled_out 和 legacy risk
维作为提示，不把 risk tags 重新提升为分母。

结论：接受为显式剩余风险。

### 8. Report 交付兼容

receipt 对 canonical report 的强制只由 manifest 的
`canonical_report_required=true` 激活；旧 manifest 仍可验证，Engine/Wrapper 新 run
必须显式开启。invalid/incomplete run 不得遗留 `final_report.md`。

结论：通过。

## 最终结论

**APPROVE FOR IMPLEMENTATION**。

批准范围：

- threat model schema/validator/compiler；
- Direct threat-mode、degraded legacy 状态和 threat coverage；
- wrapper threat-plan freeze；
- model provenance；
- canonical finalizer report 与 receipt binding；
- 对应 validator、测试、文档和版本升级。

不批准：

- 根据 8180 oracle 编码端点特判；
- 引擎自动伪造 threat；
- Direct 自建可信 authority；
- 动态扩大 trusted frozen plan；
- 记录任意模型 settings/环境变量；
- 跳过失败先行测试或全量回归直接发布。
