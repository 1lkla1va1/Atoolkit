# Atoolkit v9.0 修复方案反向审核（2026-07-18）

审核对象：`design/迭代方案/迭代方案v9.0_完全体结果归因与跨Run续航.md`

## 第一轮裁定：REJECT

若把“完全体”直接实现成自动扩张当前 Run、自动接受旧报告或让 incomplete Run 提交全部 inventory/negative，会产生六个 P0/P1 风险：

1. 动态 discovery 原地扩张 denominator，覆盖率不可复验；
2. miss attribution 被误当成 closure，给虚假阴性换一个名字；
3. rejected Finding 进入 ProjectState，污染 root registry；
4. incomplete Run 的模型自由文本 Intent 被跨 Run 信任；
5. submission checker 只看文件名，不验 receipt/hash/authority，手工报告仍可伪装；
6. 为实现跨 Run 续航放宽 Direct authority，破坏 v8.9 安全边界。

初始方案因此不得施工。

## 修订要求与复核

### 1. Attribution 只解释，不闭格

每个 cell 的终态仍只来自 accepted Finding、深阴性或结构化 not-applicable。归因器对 open 状态只能输出原因与下一动作；遇未知/歧义状态必须 `unexplained + fail closed`。

结论：通过。

### 2. Continuation 必须 Host 派生且最小提交

incomplete Run 只允许提交确定性生成的 continuation；Finding 仍需 proof pass，negative/dead-end 仍需完整 closure。continuation 不作为漏洞、阴性或覆盖率，只是下一轮 pending work。

结论：通过。

### 3. Frozen denominator 不变

新 endpoint/param 继续只进 next-run agenda；Scheduler 在下一独立 Run 才消费。当前 Run 的 cell 数、闭合率和报告不因 discovery 改写。

结论：通过。

### 4. Submission 必须验 authority 链

正式提交检查必须重验 delivery、receipt、authority anchor、canonical report hash、attribution、closure 与 secret scan。无 manifest/receipt 的 `final_report.md` 明确标 orphan，不能因内容看起来完整而放行。

结论：通过。

### 5. 垃圾风险门不得只靠标题关键词

保留现有 blacklist，但增加结构化结果门：500/type error 必须证明安全边界后果；credential disclosure 必须证明跨边界获取或实际未授权使用。假设性 ATO 继续只留 chain hypothesis。

结论：通过。

### 6. Legacy audit 只读且不升级真值

审计允许读取旧 Markdown 和 Finding 包，但不得创建 manifest、coverage、accepted Finding 或 ProjectState commit。输出只列文件路径、reason code、计数和敏感信息种类，不回显秘密。

结论：通过。

### 7. Authority 边界不放宽

当前 bundled POSIX wrapper/Direct 继续 `authority_trusted=false`。v9 只能在已有可信 supervisor attestation 时提交 continuation 和 verified delivery；没有该能力时仍返回 incomplete diagnostic。

结论：通过。

### 8. 模型独立性

provider/model 仍只是 manifest provenance。归因、续航、proof、submission 和 secret gate 全部是 Python 确定性代码，不依赖特定模型措辞。

结论：通过。

## 最终批准范围

**APPROVE FOR IMPLEMENTATION**，严格限定为：

- per-cell miss attribution 与 next-run agenda；
- Host-derived continuation 的最小跨 Run 提交及 scheduler 消费；
- receipt-bound submission checker 和 legacy/direct audit；
- structured garbage-result policy；
- manifest/receipt 版本门、测试、文档和 9.0.0 版本升级。

不批准：

- 自动接受或迁移旧 Markdown Finding；
- 当前 Run 动态新增 threat/cell；
- attribution 直接关闭 coverage；
- incomplete Run 提交模型阴性/dead-end；
- 放宽 Direct authority 或声称当前 backend 已有可证明 containment；
- 根据某个靶场/SRC 的具体域名、端点或漏洞写特判。

施工完成后必须重新执行反向验收；任一不批准项出现即停止发布。
