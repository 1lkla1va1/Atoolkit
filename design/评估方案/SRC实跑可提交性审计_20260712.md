# SRC 实跑可提交性审计（2026-07-12）

## 结论

本轮按“是否可直接提交且能经受厂商初审”审核 `runs/zhujie100` 与 `runs/aisechub`，不使用靶场 oracle，不以 finding 数量代替质量。

- `aisechub`：3 个落盘 finding.json，严格证明合同通过 0 个。
- 合计：0/13 submission-ready。

这与用户反馈一致：当前主要失败模式不是“没输出”，而是把可疑现象、正常公开能力或待验证利用链过早写成了漏洞。

## 人工可提交性分层

### A：可直接提交

0 条。

### B：保留为候选，必须重新取证

1. X-Forwarded-For 限频差异：需证明同一保护策略、稳定阈值差分与实际安全边界；不得写“无限暴破/ATO”。
2. 用户枚举：需用同状态的已知有效/无效账号多次对照，不能用“已锁定账号 vs 随机账号”直接外推。
3. HR 卡片列表：需先证明它不是公开任务市场/正常访客内容，并证明合同 URL 或字段存在真实非公开边界。
4. 空 Body 自动授权：需有授权前/后状态和实际数据同步结果，接口名和成功文案不能替代状态影响。
5. 任务生成：需有队列/资源 before-after 与有限次数的持久影响；“可多次调用”不等于资源耗尽。
6. 邮箱修改：当前只证明已登录用户修改自己的邮箱；只有完成密码找回并以新攻击者会话接管身份，才可声称 ATO。

### C：按当前根因/影响应拒绝

1. 标签树匿名可读：业务分类体系属于典型公众内容，未证明任何访问预期。
2. 邀请活动详情：活动代码、奖励比例和说明更接近公开活动配置，后续奖励绕过也已被现有证据阻断。
3. 类型混淆触发 500：单请求 500 且无持久可用性影响、无注入结果，不是可提交漏洞。
4. 不存在的消息 ID 返回 success：未证明其他用户真实消息状态发生变化。
5. IM Token 返回：未证明 Token 可越权连接、读取或发送消息，当前更像前端正常获取自身凭证。
6. Frontier Token 返回：未找到服务端点且未证明 Token 可用，不能把字符串出现写成内部服务访问。
7. LabelGPT JWT 返回：短期委派 Token 与 JWT 解码字段本身不是漏洞，未证明越权下游能力。

## 共性根因

1. **缺访问预期**：匿名 200、两个账号看到相同内容就被标未授权/IDOR，没有先判断资源是否本来公开。
2. **影响跳级**：邮箱可改、Token 出现、限频差异被直接升级为 ATO、横向移动、无限暴破或大量泄漏。
3. **只有现象包**：多数 finding 只有单个响应，缺 request、正常对照、跨身份差分、owner/victim 回查和持久状态变化。
4. **链路冒充根漏洞**：`chain_feasible=true` 同时还写着 blocker，说明最终影响尚未发生，却进入标题和 severity。
5. **低质量类别未前置过滤**：500、正常 Token 下发、公开枚举类内容没有在候选阶段被淘汰。

## 已落地的通用修复

- `未授权/unauth/auth-bypass` 统一进入 authorization proof profile，不再回退到 generic response diff。
- 授权类 finding 必须提供 `verification.access_expectation`，证明 `owner_only`、`authenticated_only`、`role_restricted` 或 private 边界。
- `same_endpoint_denial` 必须是同一归一 API 路径的原始 401/403，不能拿无关接口的拒绝响应充数。
- 匿名未授权必须附 public baseline；若资源标记能从公开入口正常发现，直接拒绝。
- RCE、ATO/会话接管、批量数据分别要求执行 nonce、身份标记、有限记录数。
- Race 的 success 数必须能从 raw concurrency log 中复算。
- legacy Markdown、chain hypothesis、proof-ready candidate 均不能 accepted、计分或关闭 Coverage Cell。

## SRC 后续验收指标

下一轮不看“报告数量”，看以下指标：

- submission-ready precision：抽审 accepted finding 中可直接提交的比例，目标 ≥ 80%。
- public-as-unauthorized false positive：目标 0。
- proof-confirmed root findings / 10 turns：衡量有效深度，不统计 chain 文案和 impact 重复项。
- 高价值开放面：认证、交易、对象授权、文件/输入类未闭合参数数应持续下降。
- 复验一致性：同一 finding 用原始包/PoC 重放时结论一致，且不存在 blocker 与 proven impact 冲突。
