# Atoolkit v8.10 修复验收报告（2026-07-16）

## 1. 验收结论

**PASS，可以发布。**

v8.10 已完成“方案 → 反向审核 → 失败先行测试 → 实施 → 分层复检”闭环。它修复的是 v8.9 在 QoderWork/Direct 实测中暴露的执行反馈断链，不宣称已经成为理论上的“完全体”，也不承诺真实 SRC 必然新增漏洞。

当前准确定位：

- Engine/Reporting 的 proof、authority、cross-Run 真值和 finalizer 基础继续保持 v8.9 的 fail-closed 边界。
- Direct/QoderWork 从“只有文字方法论”升级为可运行的 diagnostic init/observe/checkpoint 执行层。
- WAF、无数据、过期会话和错误对象等无效实验不能再靠增加向量数闭成深阴性。
- 多 agent 不再靠主 agent 手工复制 Markdown 汇总，冲突进入机器 checkpoint。
- Direct 仍是 `authority_trusted=false / delivery_eligible=false`，没有伪造可信交付。

## 3. 复检中新增发现与修复

### 3.1 相对 inventory CLI 失败

首次真实 CLI smoke 使用仓库相对路径时，safe read 把父目录重复拼接，报 `No such file or directory`。已改为先解析绝对路径，再在该绝对父目录下执行 no-follow read；新增相对路径回归。

### 3.2 Direct queue 输出失控

旧 shop inventory 展开为 241 个 exact cells，初版 CLI 一次打印全部 knowledge hint，输出超过 200 KB，违背按需注入目标。已改为滚动 Top-16 queue，每格 hint 最多 4 张卡；完整分母仍保留在 coverage ledger，checkpoint 后自动补下一批。CLI stdout 只打印统计、queue 数量和文件路径。

## 5. 仍然不是“完全体”的部分

1. **可信 containment 未交付**：内置 Codex/wrapper 仍无法证明 `setsid()` 后代已全部静默，所以不能可信改写跨 Run ProjectState。需要真实 cgroup/job/container supervisor attestation。
2. **真实 SRC 增益尚未统计显著**：本轮证明合同和执行闭环正确，不证明一定提高平台 accepted 数。仍需固定账号/范围/版本做 3–5 轮全新 A/B Run。
3. **target fingerprint 自动 stale/reopen 未实现**：当前只记录/比较，不自动重开历史 cell。
4. **Codex 自动注入 AGENTS 仍约 453 行**：Qoder/Direct 已有短 hot path 和按需卡；彻底缩短 Codex 根 AGENTS 会改变安全边界注入，需独立基准后再做。
5. **模型探索能力仍是上限**：机器层可以阻止错误收口、提醒补测和同步状态，不能保证模型创造出有效 WAF 绕过或复杂业务链。

因此，本次可称为“v8.9 实测问题的工程闭环版”，不能称为终局完全体。下一阶段优先级应是可信 supervisor 与新的盲测 A/B 数据，而不是继续往核心提示词堆漏洞技巧。
