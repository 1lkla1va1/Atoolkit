# Atoolkit v8.12 修复方案反向审核（2026-07-17）

审核对象：`design/迭代方案/迭代方案v8.12_Engine两阶段威胁计划与SRC运行真实性.md`

审核依据：Atoolkit v8.11 代码合同、ByteDance 历史 Run 的离线产物、authority/finalizer
不可变性规则。审核只分析本地文件，没有向历史目标发包。

## 第一轮结论：REJECT

初稿构想存在六个阻断问题：

1. 若在同一个 Run 内先生成 threat 再补 manifest，会违反“首个模型动作前已有 manifest/run plan”；
2. 若 Planning 直接读取原 Recon，会把 API Key、Cookie 和个人信息再次送入模型与计划产物；
3. 仅用 identity label 计数，`owner`/`peer` 可以指向同一 Cookie，仍会复现共享会话假阴性；
4. 若 threat cells 继续调用 `seed_matrix()`，仍会被 `_classes_for_endpoint()` 扩成漏洞类笛卡尔积；
5. 只在提示词里说 scope，模型仍可把同 origin 跨模块 API 纳入当前 Run；
6. 把全部 CSRF 加入垃圾洞清单会误杀真实跨站状态改变漏洞。

因此初稿不得施工。

## 修订复核

### 1. 双 manifest 的 authority 顺序

修订采用 sibling Planning/Attack Session。Planning 的空 denominator run plan、输入 artifact hash
和 manifest 在 planning adapter 前创建；Attack 的 compiled denominator、parent hash 和 manifest 在
attack adapter 前创建。

结论：通过。

### 2. Planning 数据最小化

修订要求 Host 复制有界、no-follow Recon snapshot，并在写入 Planning workspace 前脱敏；manifest
绑定脱敏副本和原始 digest。本轮 `cookies.txt` 在 Planning 完成后才落盘，Planning adapter 网络
固定关闭。

剩余风险：通用正则不可能识别所有自定义秘密格式。控制措施是高置信 key/header 检测、输出再次
扫描、文件/字节上限和 Canonical report 二次脱敏。接受为显式剩余风险。

补充限制：当前 workspace sandbox 不证明同用户全局只读隔离。上述延迟落盘、Planning 禁网和输出
扫描是风险收敛，不是容器级 read isolation；未来需独立容器/VM/受证明 mount namespace 才能关闭。

结论：通过。

### 3. 身份真实性

修订不信任 label 数量，而比较 credential SHA-256；同值多个 label 只算一个上下文。Threat
显式声明 identity requirement，unready cell 不得 terminal。marker 只存 hash，不保存原 PII。
复核实现时进一步发现“Host 判定 ready、Attack 模型却拿不到 `--identity` 原始 header”的缝隙；
最终方案要求在 Planning 完成后生成权限收紧的 `identities.json`，按 label 提供给 Attack；finalizer
开始前的崩溃恢复对其重新计算指纹并与 frozen readiness 比较。该文件不进入 Planning 或 Canonical report。

剩余风险：不同 Cookie 不必然代表不同账号。涉及跨账号阴性仍需请求/响应中的身份与对象断言，
现有 proof/negative envelope 必须继续执行，identity readiness 只是前置门而不是漏洞证明。

结论：通过。

### 4. 唯一 coverage 分母

修订新增 `seed_threat_cells()`，每个 compiler row 精确生成一个 cell；threat 模式禁止调用通用
`_classes_for_endpoint()`。Run plan 同时冻结 surface/feature/threat identity。

结论：通过。

### 5. Scope path 而非同源授权

修订把 base/allow/deny path 同时交给 planner 和 deterministic validator。跨模块路径只能作为
unassigned/pending scope amendment，不能进入本轮 threat cells。

结论：通过。

### 6. CSRF 因果门

修订没有把 CSRF 整类屏蔽，而是新增 `cross_site_state_change` 合同，要求 before/request/after、
跨来源、可执行跨站发起载体和可复算状态变化。缺 token、手工伪造 Origin/Cookie，或暴露 token 但
无跨站结果都会失败，真实改密/转账类 CSRF 可通过。

结论：通过。

### 7. Parent lineage 的循环与替换风险

Attack 只允许固定 `<sid>.planning` parent，parent 必须位于同 authority manifests 根、project/target/
scope 相同、phase=planning、hash 匹配。Parent 不引用 child，不存在循环。authority copy 为 create-only。
Attack 还逐个绑定晋升后的 Recon snapshot，终态递归执行 parent artifact binding，避免只验证 parent
JSON 却放过 Planning 输入被替换。

结论：通过。

### 8. Resume 语义

风险：resume 时重新规划会产生不同 threat 分母。

修订要求：已有 Attack manifest/run plan 且 finalizer 尚未开始时，只验证原 parent 与 planning artifacts，
不重新调用 planner；缺少 lineage artifact 时 fail closed。一旦 finalization journal 已存在，该 sid 的
输入快照已冻结，不得继续模型执行，必须新开 sid。新的 Recon 或身份条件同样必须新开 sid。

结论：通过。

### 9. 动态发现与局部重建模

本版 Attack 中的新 endpoint 仍只能进入 next-run discovery/Intent，不能修改本轮 denominator。
“页面触发后的动态局部重建模”会创建新 sibling planning+attack Run，而不是原地改计划。

结论：通过。

### 10. ByteDance 历史问题覆盖度

| 历史问题 | v8.12 控制 |
|---|---|
| 无 manifest/plan/ledger/evidence | 两阶段 authority + exact threat cells；旧报告只能迁移为候选 |
| 共享浏览器身份却下安全结论 | credential 去重 + identity requirement + unready 不得闭格 |
| API 名称而非 METHOD/path/param | threat plan 必须绑定 canonical inventory |
| Managed Agents/支付/IAM 等漏测 | feature 全 inventory 归属 + discovery adequacy + open threat 可见 |
| 同域跨模块范围含混 | frozen path scope + scope amendment |
| 原始 Key/Token/PII 进入报告 | Planning snapshot 与 Canonical renderer 双重脱敏 |
| CSRF 现象型垃圾 P3 | 专用 cross-site state change proof contract |
| 报告自称错误模型 | manifest execution provenance 为唯一来源 |

结论：设计能够覆盖根因，不按具体厂商 endpoint 做特判。

## 最终结论

**APPROVE FOR IMPLEMENTATION**。

批准范围：

- Engine sibling Planning/Attack 与 schema-4 lineage；
- Recon 脱敏 snapshot 和 discovery adequacy；
- exact threat seeding、identity readiness 和 identity-aware closure；
- path scope gate；
- CSRF 因果证明合同；
- Canonical report 脱敏；
- CLI、测试、文档和版本升级。

不批准：

- 针对 ByteDance、8180 或任何 oracle 编码 endpoint 特判；
- Planning Session 访问目标网络；
- 缺 Recon 时隐式退回 live legacy；
- 同一 Run 动态改写 frozen denominator；
- 把 credential 原值写入 identity/planning/report；
- 只检查 identity label 不检查 credential distinctness；
- 把所有 CSRF 一律屏蔽；
- 审核通过后跳过失败先行测试或全量回归直接发布。
