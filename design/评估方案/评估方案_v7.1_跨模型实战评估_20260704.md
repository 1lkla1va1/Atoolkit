# 评估方案 v5 -- Atoolkit v7.1 跨模型实战评估与迭代路线

> 版本：v5。基于 v7.1 技能 QoderWork 驱动实测 4060/4800 分数据，对技能包做全维度评估。
> 记录时间：2026-07-04
> 评估对象：runs/shop_pentest/ 全套产物（20 命中 / 4 漏测 + 16 份 finding 包 + 覆盖台账 + summary.json + scoring_report.md）
> 对标基线：靶场官方 24 项 / 4800 分 + v4 评估 2410 分基线 + v2 评估 640 分基线
> 驱动模式：QoderWork 直接驱动（4 并行 subagent: auth/txn/idor/input），Skill Mode

## 0. 一句话结论

v7.1 将零先验配置内得分从 v7.0 的 2410 提升到 4060（+68.5%），命中率从 50.2% 跃升至 84.6%。v7.1 迭代方案中规划的 10 个改进项有 8 个在本次实测中生效，SQLi LIKE 注入假阴性归零、认证端到端链验证生效、Phase 0 非 API 页面扫描补回了商户面板和 AK/SK 泄露。但仍有 4 个配置内漏洞未命中（740 分），暴露出 WAF 绕过能力缺失、上传参数覆盖不全、重定向深度验证不足三个新盲区。技能包对得分的边际贡献约 35-40%，模型自身能力贡献约 60-65%。在 Codex/Claude/zcode 跨模型场景下，预估得分方差约正负15%，技能包的 checklist 粒度是跨模型通用瓶颈。

### 5.3 跨模型通用的瓶颈

不论哪个模型执行，以下问题都会出现：

1. WAF 绕过能力缺失（影响所有模型）：技能包没有 WAF 对抗指导，任何模型遇到"检测到非法关键字"后都会放弃。

2. 上传接口非文件参数覆盖不全（影响所有模型）：技能包的参数穷举规则集中在交易端点，未覆盖文件上传类接口的 category/dir/path 等元数据参数。

3. 存储型 XSS 验证深度（影响中等遵从度模型）：技能包要求"返回空/异常后检查存储效果"，但规则不够强制。

4. use_points/积分参数的变体测试（影响所有模型）：技能包提到"参数名变体"但没给出具体 payload 列表。

结论：740 分的漏测中约 560 分是技能包规则粒度问题（跨模型通用瓶颈），180 分是 WAF 能力问题（也是跨模型通用瓶颈）。

## 6. SRC 实战适用性评估

### 6.1 靶场 vs 实战差异分析

| 维度 | 靶场特征 | SRC 实战特征 | 技能包适应性 |
|---|---|---|---|
| 目标规模 | 6 页面 + 48 API | 50-500 页面 + 数千 API | Phase 0 需扩展 |
| 漏洞密度 | 24/48 = 50% | 通常 1-5/数百 = 小于2% | 需调整深度优先策略 |
| 认证复杂度 | 简单 SHA256 + 固定账号 | OAuth/SSO/2FA/设备指纹 | 认证面规则需扩展 |
| WAF/IDS | 无 | 几乎必有 | 需增加 WAF 对抗 playbook |
| 技术栈 | PHP + MySQL | Java/Go/Node/Python + 各种 DB | 技术栈分支需扩展 |
| 前端框架 | 原生 JS + 服务端渲染 | SPA (React/Vue/Angular) | Phase 0 需支持 SPA 路由分析 |
| 测试时间 | 无限制 | 通常 1-3 天 | 需增加时间预算分配规则 |
| 报告要求 | 内部使用 | SRC 平台审核 | 七问验证门已覆盖 |

### 6.2 实战场景下的预估效果

| 能力 | 预估覆盖率 | 说明 |
|---|---:|---|
| 认证/授权测试 | 80% | 技能包最强方向 |
| IDOR/越权 | 75% | role-pair 覆盖完善 |
| 业务逻辑 | 65% | 参数穷举有效，复杂业务规则需人工补充 |
| 注入类 (SQLi/XSS/SSRF) | 55% | 基础注入有效，WAF 绕过不足 |
| 文件上传/路径穿越 | 50% | 基础绕过有效，深度绕过不足 |
| 链式利用 | 35% | 通用链式框架缺失 |

综合评估：在 SRC 实战中，技能包能覆盖中级渗透测试人员约 60-70% 的工作流。

每次版本迭代后的评测应遵循：
1. 重置靶场数据
2. 新 session（无先验记忆）
3. 执行 Phase 0 侦察
4. 并行渗透测试（4 agent）
5. 复测阶段（完整性门推动补测）
6. 生成 summary.json + final_report.md
7. 与 vuln_config.php 对照评分
8. 填写 8 维度评分表
9. 归因分析（新增命中 / 退化项 / 漏测项）
10. 更新纵向对比表

## 9. 总结与建议

### 9.1 v7.1 的核心成就

1. 零先验得分 +68.5%：从 2410 到 4060，证明 v7.1 的三大支柱有效。
2. 绕过验证 + 越权访问 100% 覆盖：从 v2 的近乎空白到完全覆盖。
3. 机制有效率从 73% 提升到 90%：迭代投入产出比优秀。
4. 假阴性从 3 个降到 1 个：SQLi LIKE 和文件上传路径穿越假阴性均已消除。

### 9.2 v7.1 的核心教训

1. 参数穷举规则必须具体到 payload 级别："测试 use_points"不够，必须说"测试 use_points/use_point/points + 负数/精度/溢出"。
2. WAF 绕过是技能包的下一个天花板：靶场有一个端点配了关键字过滤就挡住了 200 分。
3. 存储型漏洞验证需要"POST 后 GET 验证"的闭环。
4. 技能包贡献度有上限：约 35-40% 来自技能包，60-65% 来自模型自身能力。

### 9.4 对技能包作者的寄语

Atoolkit 从 v2 到 v7.1，四轮评测走过了一条清晰的进化路径：640 -> 2410 -> 4060。每一步都有数据支撑的归因和可量化的改进。这不是一个"改了就祈祷"的项目，而是一个"改了->测->量化->再改"的工程化项目。

技能包的价值不在于让模型"变聪明"，而在于给模型装上纪律和清单——防止它"找到一个洞就深挖忘了横向扫描"、"WAF 拦截就放弃"、"返回空就当安全"。这些纪律的价值在 SRC 实战中比靶场得分更重要。

下一个里程碑是 v8：WAF 绕过 + 链式利用 + 跨模型兼容。如果 v7.1 证明了"规则粒度"是跨模型通用瓶颈，v8 应该证明"知识深度"是下一个突破口。
