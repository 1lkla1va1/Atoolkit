## 0. 一句话结论

**AI 已经会做渗透，我们要做的不是教它方法，而是给它一个运行平台 + 三类约束（边界 / 报告标准 / 防失控），然后按"问题的可验证性"决定放手多少。** SRC 挖掘是这条光谱的中段：通用洞放手，逻辑洞半控，长会话靠工程兜底。

## 1. 底层认知：为什么这套打法成立（知识库社区 3「模型能力形成与约束对齐」）

来自《AI 真的"懂"安全吗？》的核心论断，是整套设计的地基：

- **安全能力是通用 Agent 能力的投影**——没有厂商专门训练"网络安全"，他们训练的是 Agent / 可验证环境 RL / 长程推理。安全能力是这套训练在攻防场景的间接投影（"练钢琴的手指学吉他更快"）。
- **四层能力形成路径**：预训练（知道）→ SFT+专家蒸馏（理解）→ 可验证环境 RL（试错）→ Agentic（行动）。**安全能力真正涌现在第三层之后**。
- **三条推论，直接决定 SRC 系统怎么搭**：
  1. 安全知识模型早就够了，**真实瓶颈是"长程自主执行"**——复杂环境里持续推进、失败后自主换路线。→ 所以系统重心放在"维持长会话不崩"，而非"教它漏洞知识"。
  2. **可验证环境的密度 > 参数规模**。→ 对 SRC 的迁移：给 AI 一个能"当场验证危害"的闭环（重放 PoC、对比响应）比换更大模型更有效。
  3. **没有全场景通吃的模型，只有适配任务的模型**（社区 8「模型选型方法论」）。→ 按任务拆解选模型：粗筛用便宜模型跑量，精验/裁决用强模型。

> **设计含义**：你的系统不是在"补足模型不懂安全"，而是在"喂它一个能长程执行 + 当场验证 + 不跑偏"的脚手架。

## 2. 主梁：三份"约束哲学"是同一件事（社区 6「边界哲学与报告标准」）

知识库里 betweenness 最高的桥接节点 `不束缚AI只设边界` 把三份文档连成一体——它们讲的是同一条主梁：

| 来源 | 表述 | 本质 |
|---|---|---|
| 设计指南 | 不束缚 AI，只设边界 | 解放推理，约束行为 |
| 六位数赏金 | 约束优先于教学；外部验证优先于自检 | 同上 + 强调外部强制 |
| Web3 | 约束引导模型而非自由生成；模型负责推理我们负责定义边界 | 同上 + 强调可验证推理路径 |

**灵魂金句（跨 4 份文档语义等价，知识库已自动连边）**：
> **现象不是漏洞，漏洞是结果。** 报的是结果（越权/注入/RCE），不是过程（信息泄露/配置）。
> ≡ 状态码 200 ≠ 业务漏洞 ≡ 模式匹配 vs 因果证明。

这就是 SRC 降噪的总开关：AI 误报的根源不是"不够聪明"，是**训练目标（相关性推断）与任务目标（因果链证明）错位**。SRC 要的是后者。

## 3. 骨架：唯一的核心技能文件（社区 0 + 社区 6，God 节点第 1/2 名）

知识库里连接度最高的两个 God 节点就是 `核心技能文件实例`（8 边）和 `六大设计准则`（7 边）——它是整个系统的灵魂，也是你**已经写好的** `核心技能文件.md`。六大准则（知识库超边 `核心技能文件六大准则`）：

1. **垃圾洞清单置顶**——CORS / sourcemap / 安全头 / Self-XSS / 无 PoC，绝对不报。放最开头对抗长对话遗忘。
   - ⚠️ 赏金文章的血泪教训：**别写"除非…则可以报"的例外**，AI 会当成"需要辩证看待"反而误报上升。砍成一句"不报，除非附带完整窃取 PoC"。
2. **灵魂金句**——现象不是漏洞，漏洞是结果（见 §2）。
3. **指挥官只装决策逻辑，payload 下沉模型**——"不教鲨鱼游泳"。不写注入语法/绕过编码，AI 全知道。
4. **防遗忘机制**——速查卡（10-20 条短句）+ Phase 切换重读（God 节点，6 边）。
5. **七问验证门**——报告前逐条自答，第 7 问"发到平台会被关掉吗"是终极过滤。
6. **决策树而非固定流程**（God 节点，6 边，全图 betweenness 最高 0.230）——按登录态/技术栈/功能/ROI 动态分支。

**关键比例**：全文 150-200 行，绝不超 300。**迭代方式**（飞轮，社区 6）：报噪音→加垃圾洞清单；越界→加铁律；报告差→强化七问；死磕→调时间约束。**这个文件是你全部的竞争力**。

## 4. 关于 Skill：钥匙不是咒语（社区 1「Skill路由与认知状态架构」）

《也许我们不需要 Skill》给了 SRC 用 Skill 的纪律。Skill 只有三个作用维度，满足任一才有价值：

| 维度 | 作用 | SRC 用法 |
|---|---|---|
| **路由** | 把注意力激活到正确知识抽屉 | ✅ 用观察到的现象当钥匙：「用户输入反射→XSS/SSTI」「大量对象ID→IDOR/BOLA」 |
| **约束** | 压缩输出空间（负面约束遵循度远高于正面建议） | ✅ 垃圾洞清单、七问门就是负面约束 |
| **补充** | 注入模型没见过的私有信息 | ✅ 你对特定目标的私有缺陷模式、内部业务 SOP |

**实证（知识库社区 1）**：SWE-Skills-Bench 测 49 个 Skill，80% 对成功率零影响、3 个反而拖累；AI 自生成 Skill 期望收益为负。**陷阱**：常识百科型 / 触发过宽 / 单文件 600 行大全（Lost in the Middle）/ 空壳包装 / 互相冲突。

**对 SRC 的结论（与逻辑漏洞文章的 `SKILL兜底与激活而非限制` 在知识库里语义等价）**：
- 你环境里那批 HackSkills（idor / business-logic / race-condition…）的正确定位是**用 BM25/FTS5 检索激活攻击面灵感的钥匙**，不是流程枷锁。
- **关键升级**：从"开局一把梭全加载"改成"**意图触发检索**"（见 §6.3），用多少加载多少。
- 日常保持 5-8 个精选，超 10 个边际递减。

## 5. 血肉：长链路逻辑漏洞怎么打（社区 4「人控建模」+ 社区 7「Loop失效」+ 社区 12）

通用洞靠 §3 的轻约束够用，但业务逻辑洞"无 Payload 特征、200≠漏洞、每次从零建模"，纯放手会越界+遗漏。补三件：

### 5.1 人控建模前置（社区 4，《AI 加速逻辑漏洞测试》）
模式：**人定方向，AI 整理→拆解→执行→验收**。
- **为什么不纯 AI 收集流量**：生产登录有二次核身/短信/验证码，人 1-2 分钟过，AI 模拟浏览器识别验证码噪声大→**人负责拿新鲜 Cookie/Session**。
- **人不能完全不懂业务**：人要做 AI 做不了的两件事——划边界不越界、验收排误报。
- 产出**任务书/待测清单**（目标 + Cookie/Session + 路径功能点），必要时让 AI 生成 **Mermaid 时序图**理清金钱流/权限流。

### 5.2 TODO vs PLAN 双模式（社区 7，对抗雪崩效应）
小错误在多轮 ReAct 中传播放大→任务失控。按链路长度选：

| | TODO List | PLAN 模式 |
|---|---|---|
| 适用 | 单字段/IDOR/短链路 3-5 步 | 跨子系统/多漏洞/复杂场景 |
| 特点 | 嵌入 ReAct、低延迟 | 前置 DAG/任务树、全局稳定 |

### 5.3 Loop 三种失效及外部应对（社区 7，《六位数赏金》）
Agent 本质是 Loop（观察→推理→行动→反馈），SRC 里三种失效——**用外部工程层兜底，别指望 AI 自律**：

| 失效 | 表现 | 外部应对 |
|---|---|---|
| 方向偏移 | 测越权跑去测 CORS | 决策树引导 + 切换前强制重读速查卡 |
| 规则遗忘 | 长会话忽视"不报"清单 | 关键规则置顶 + 20-30 轮后定时注入提醒 |
| 空循环/伪完成 | 无产出或**幻觉漏洞** | 外部计时器 20min 强制切向；超 50 轮强制总结重启 |

> 实测锚点：redhaze.top 无训练材料靶场，Qwen3.7/DeepSeek/GPT-5.5 都能在 50-85 分钟内产出 16-23 个洞、逻辑洞占比 41-81%。**有业务建模的版本深度更深、越界/遗漏更少**。

## 6. 加固：对抗长会话失控的工程机制（社区 2「链上分析与假设驱动推理」）

Web3 那份把"AI 长任务为什么会烂"拆得最透，机制直接迁移到 SRC 长会话：

### 6.1 注意力是稀缺资源，不是窗口大小
"从喂更多到喂更准"。别把整个 Burp 历史塞给 AI→注意力稀释 + 上下文漂移，模型遗忘最初约束。**按需加载**：先喂端点/参数/认证的结构摘要，深入某条链路时再调完整包。

### 6.2 认知状态架构（社区 1/2 桥接，对抗 20 轮后失忆）★ 最高 ROI 升级
- **不让模型"记住"，让系统"知道"**：当前认知快照**不存在对话历史里**，由外部程序维护一个**结构化状态对象**。
- 结构化状态对象 + 每轮全新对话 + 完整注入 Prompt → **强制每轮认知对齐**。
- 这个状态对象同时是**自带的 Audit Trail**——对 SRC 就是天然的复现链证据（每个假设的建立/推翻/修正全程可查，与知识库里 `防遗忘速查卡 ≈ 不信任LLM工作记忆` 的自动连边一致）。
- 配套 **两阶段输出（CoT 解耦）**：第一阶段自由推理（不上 JSON 枷锁，最大化推理深度）→ 第二阶段严格 schema 更新状态。"想清楚"和"说清楚"分离。

### 6.3 意图触发式上下文注入（纠偏 §4 的 Skill 用法）
检索依据不是"文本相似度"，是"**Agent 的逻辑推理意图**"。Agent 每轮主动声明知识需求（"我现在要测越权，给我 IDOR 攻击面"）再触发 Skill 检索，而非开局全塞。

### 6.4 假设驱动的双线探索（替代 Plan-and-Execute）
分析是"**数据流/权限流（宏观概率信号）+ HTTP 请求链（微观确定验证）**"双线交替推进，交汇点=漏洞点。LLM 出概率性怀疑，工具层做确定性验证（换 token 重放看响应）。类比"符号执行 × 模糊测试"。Plan-and-Execute 会锁死探索路径。

### 6.5 多模型 + Judge（社区 5「四大难点」催生，按需用）
单模型有结构性盲区 + 概率性遗漏 + **无法自验**（不知道"没发现"是"没有"还是"没看到"）。
- **SRC 取舍**：日常挖洞验证成本低，不必每会话上多模型。**关键资产终验 / 边界报告决定是否提交**时，用第二个模型做 Judge 复核 + 置信度分层很划算。
- 与赏金文章的"多层验证架构 / 谁来审核审核者"在知识库里语义等价：**AI 自检只是第一道降噪器，不是最终质量保证**。

## 7. 验证与防失控：物理证据 > AI 声明（社区 0 + 社区 11）

这是 SRC 系统不被"AI 自我表扬"骗的关键（设计指南第八/十一章）：

- **双重验证**：AI 的状态标记声明（说了什么）vs 磁盘物理证据（实际做了什么）。
- **十二条终态决策规则**：声明 VULN_FOUND 但无有效报告 → 降级 low_roi；无标记但有证据 → 补救为 vuln_found。**物理证据可推翻 AI 声明**。
- **有效报告标准**：P1/P2/P3 + 标题 + 正文≥200 字 + 含可复现证据（curl/HTTP 包）。
- **约束的强制执行**（外部层硬编码，不靠 Prompt）：输出解析器（无结构化 PoC→拒绝）、关键词拦截器（标题命中垃圾洞→拒绝）、危险命令拦截（`rm -rf`/`DROP TABLE`→直接拦）、磁盘自循环检测、API 健康指数退避。

## 9. 落地路线（对你当前进度）

你已经有 `核心技能文件.md`（成熟）+ 一批 `low_priv_*.json`（一个越权挖掘会话正在跑、已落盘结构化证据）。所以你不在阶段一，按 ROI 排：

1. **短期（最高 ROI）**：补 **§6.2 认知状态架构**——给长会话维护外部状态对象（当前假设/已验证/待测 TODO/已落盘证据），每轮全量注入。对你这类长会话"20 轮后失忆"是最高 ROI 的一步。
2. **中期**：Skill 从"开局加载"改成 **§6.3 意图触发检索**；接 **§6.5 双层模型**（便宜模型跑量 + 强模型 Judge 终验）。
3. **持续**：唯一真正重要的事——**每次实战后迭代核心技能文件**（§3 飞轮）。这是系统竞争力的全部来源。

## 10. 红线（贯穿全程，来自核心技能文件 + 铁律）

- 授权范围外不发包/不扫描/不建连；越界即停标 `NEED_INPUT`。
- 不做破坏性操作（删库/改数据/DoS/压测爆破）、不横向扩散、不外传数据。
- 不从截图读凭据自行登录（遇登录墙→停）。
- 验证危害到"足以证明"即止，不做超出取证所需的进一步利用。
- 写文件只写指定临时目录，禁止写入自身读取路径子目录（防自循环）。

> 关于"拒答解法"（社区 10）：合法化上下文 + 术语重构是为了让模型在**已授权**测试中不无谓拒答，**绝不等于**绕过红线去打未授权目标。授权边界是硬约束。
